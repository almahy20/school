# تحليل شامل لمشروع إدارة المدارس (School Master)

هذا المستند يقدم تحليلاً كاملاً للهيكلية الحالية للمشروع، التقنيات المستخدمة، والصفحات المتوفرة، لمساعدتك في عملية إعادة الهيكلة المستقبلية.

## 1. نظرة عامة على المشروع
المشروع عبارة عن منصة SaaS (Software as a Service) لإدارة المدارس بنظام تعدد المستأجرين (Multi-tenancy)، حيث يمكن لكل مدرسة الحصول على بيئتها الخاصة وإدارة طلابها، معلميها، وأولياء الأمور.

## 2. المكدس التقني (Technical Stack)

### **الواجهة الأمامية (Frontend)**
- **الإطار الأساسي:** React 18.3 مع Vite كمحرك بناء.
- **لغة البرمجة:** TypeScript لضمان سلامة الأنماط (Type Safety).
- **التنسيق (Styling):** Tailwind CSS مع استخدام مكتبة Shadcn/UI (مبنية على Radix UI) للمكونات الجاهزة.
- **إدارة الحالة والبيانات:** TanStack Query (React Query) v5 للتعامل مع طلبات البيانات والتخزين المؤقت (Caching).
- **التوجيه (Routing):** React Router DOM v6.
- **النماذج (Forms):** React Hook Form مع Zod للتحقق من البيانات (Validation).
- **الإشعارات:** Sonner و Radix Toast.
- **الأيقونات:** Lucide React.

### **الخلفية وقاعدة البيانات (Backend & Database)**
- **Supabase:** يُستخدم كمنصة متكاملة (Backend as a Service) تشمل:
    - **PostgreSQL:** قاعدة بيانات علائقية.
    - **Supabase Auth:** لإدارة المستخدمين وتسجيل الدخول (Email/Password).
    - **Supabase Edge Functions:** لتنفيذ الأكواد في جهة الخادم (مثل إرسال الإشعارات).
    - **Realtime:** لتحديث البيانات فورياً في المتصفح.
    - **Storage:** لتخزين الملفات والصور (مثل شعارات المدارس).

### **تقنيات إضافية**
- **PWA (Progressive Web App):** يدعم التطبيق التثبيت على الهواتف والعمل أوفلاين جزئياً باستخدام `vite-plugin-pwa` و `Service Workers`.
- **Push Notifications:** دعم إشعارات الدفع باستخدام Web Push API.
- **التدويل (i18n):** التطبيق يدعم الواجهة العربية (RTL) بشكل أساسي.

## 3. هيكلية الصفحات والمسارات (Routes)

### **أ- الصفحات العامة (Public Routes)**
- `/home`: الصفحة الهبوط (Landing Page).
- `/login`: صفحة تسجيل الدخول.
- `/register/teachers/:school_slug`: تسجيل معلمين لمدرسة معينة.
- `/register/parents/:school_slug`: تسجيل أولياء أمور لمدرسة معينة.
- `/payment/:orderId`: صفحة الدفع والاشتراك.

### **ب- صفحات لوحة التحكم (Dashboard)**
- `/`: المسار الرئيسي، يوجه المستخدم حسب دوره (Admin, Teacher, Parent).
- `/waiting-approval`: صفحة الانتظار حتى يتم تفعيل الحساب من قبل الإدارة.
- `/onboarding`: خطوات الإعداد الأولية للمدارس الجديدة.

### **ج- صفحات الإدارة (Admin Only)**
- `/super-admin`: لوحة التحكم العليا للنظام بالكامل.
- `/students`: إدارة الطلاب (إضافة، تعديل، حذف).
- `/students/:id`: تفاصيل الطالب الشخصية والأكاديمية.
- `/teachers`: إدارة المعلمين.
- `/teachers/:id`: تفاصيل المعلم.
- `/parents`: إدارة أولياء الأمور.
- `/parents/:id`: تفاصيل ولي الأمر.
- `/classes`: إدارة الفصول الدراسية.
- `/classes/:id`: تفاصيل الفصل (الطلاب، المنهج، الحضور).
- `/fees`: إدارة الرسوم الدراسية والمدفوعات.
- `/users`: إدارة مستخدمي النظام وصلاحياتهم.
- `/data-retention`: إعدادات الاحتفاظ بالبيانات.
- `/manage-complaints`: إدارة شكاوى أولياء الأمور والرد عليها.
- `/admin-reports`: التقارير الإدارية والإحصائيات.

### **د- صفحات المعلمين (Teachers)**
- `/attendance`: تسجيل غياب وحضور الطلاب.
- `/classes`: عرض الفصول المسندة للمعلم.

### **هـ- صفحات أولياء الأمور (Parents)**
- `/parent/children/:id`: الملف الشخصي للابن/الابنة.
- `/parent/children/:id/grades`: عرض الدرجات والنتائج.
- `/parent/children/:id/attendance`: عرض سجل الحضور والغياب.
- `/parent/children/:id/financial`: عرض الموقف المالي والرسوم.
- `/parent/children/:id/curriculum`: عرض المنهج الدراسي.
- `/complaints`: تقديم شكوى جديدة.

### **و- صفحات مشتركة**
- `/notifications`: مركز الإشعارات.
- `/messages`: نظام المراسلة الفورية.
- `/settings`: إعدادات الحساب والملف الشخصي.
- `/expired`: تنبيه عند انتهاء اشتراك المدرسة.

## 4. هيكلية قاعدة البيانات (أهم الجداول)
1. **`schools`**: بيانات المدارس، الشعارات، وإعدادات العلامة التجارية.
2. **`profiles`**: بيانات المستخدمين (الاسم، الدور، رقم الهاتف، المعرف الخاص بـ Auth).
3. **`students`**: بيانات الطلاب وربطهم بالفصول وأولياء الأمور.
4. **`classes`**: الفصول الدراسية وربطها بالمعلمين والمناهج.
5. **`attendance`**: سجلات الحضور والغياب اليومية.
6. **`exam_templates`**: قوالب الامتحانات والدرجات.
7. **`grades`**: درجات الطلاب في المواد المختلفة.
8. **`fees` & `payments`**: إدارة الشؤون المالية والرسوم.
9. **`notifications`**: سجل الإشعارات المرسلة للمستخدمين.
10. **`complaints`**: سجل الشكاوى بين أولياء الأمور والإدارة.

## 5. توصيات لإعادة الهيكلة المستقبلية
- **استخدام Next.js:** للانتقال من CSR (Client Side Rendering) إلى SSR/ISR لتحسين الأداء و SEO في صفحات معينة.
- **تطوير نظام الصلاحيات (RBAC):** استخدام سياسات RLS (Row Level Security) في Supabase بشكل أكثر صرامة.
- **تجزئة المكونات:** إعادة توزيع المكونات في مجلدات وظيفية (Features-based architecture) بدلاً من النوع (UI, Components).
- **تحسين الأداء:** استخدام `Server Actions` بدلاً من الاعتماد الكلي على Client-side fetching إذا تم الانتقال لـ Next.js.
- **واجهة برمجة تطبيقات (API):** توثيق API الخاص بـ Edge Functions لاستخدامه مستقبلاً في تطبيقات الموبايل (Native).
