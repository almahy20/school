# دليل صيانة وترقية النظام (Maintenance & Upgrade Guide)

هذا الدليل مخصص لضمان استمرارية تشغيل النظام بنسبة 99.9% وتوضيح آليات الاسترداد التلقائي.

## 1. آليات الموثوقية (Reliability Mechanisms)

### 1.1 استرداد الأخطاء (Auto-Recovery)
- **Error Boundaries**: يستخدم النظام `GlobalErrorBoundary` في [App.tsx](file:///f:/schoole/school-main/src/App.tsx) لمنع تعليق التطبيق بالكامل عند حدوث خطأ في مكون فرعي.
- **Retry Logic**: تم تكوين `QueryClient` لإعادة محاولة طلبات البيانات تلقائياً حتى 3 مرات عند حدوث أخطاء في الشبكة.

### 1.2 مراقبة الصحة (Health Monitoring)
- **HealthMonitor**: مكون يعمل في الخلفية لمراقبة حالة الاتصال بالإنترنت والوصول لقاعدة بيانات Supabase.
- **التنبيهات**: يظهر شريط تنبيه علوي فور انقطاع الاتصال مع خيار "إعادة المحاولة" يدوياً.

## 2. إجراءات الصيانة (Maintenance Procedures)

### 2.1 التحديثات الدورية
- **قاعدة البيانات**: يجب مراجعة ملفات الهجرة (Migrations) في `supabase/migrations` قبل تطبيقها على بيئة الإنتاج.
- **Service Worker**: عند تحديث الكود المصدري، سيقوم الـ Service Worker بتحديث نفسه تلقائياً (Stale-while-revalidate).

### 2.2 النسخ الاحتياطي (Backup)
- يتم النسخ الاحتياطي لقاعدة البيانات تلقائياً عبر منصة Supabase.
- يمكن العثور على النسخ اليدوية في مجلد `supabase/backup`.

## 3. اختبار الاستقرار (Stability Testing)

لضمان عمل النظام تحت ضغط عالٍ:
1. تأكد من أن الـ Service Worker يقوم بتخزين الأصول الأساسية (Assets).
2. اختبر سلوك التطبيق عند تفعيل "Offline Mode" في المتصفح.
3. راقب استهلاك الذاكرة عبر متصفح Chrome (Performance Tab).

## 4. سياسة الترقية (Upgrade Policy)
1. قم دائماً بتشغيل `npm run build` للتأكد من خلو الكود من أخطاء التجميع.
2. اختبر الميزات الجديدة محلياً باستخدام `npm run dev`.
3. قم بتوثيق أي تغييرات في هيكل قاعدة البيانات فوراً.
